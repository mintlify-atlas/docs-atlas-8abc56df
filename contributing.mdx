Kaku is built in Rust on top of the WezTerm engine. The development setup requires a recent Rust toolchain, a small set of Cargo tools, and a macOS host. Once the environment is ready, the `make dev` command gives you a fast edit-compile-run loop without any packaging overhead.

## Setup

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/tw93/Kaku.git
    cd Kaku
    ```
  </Step>

  <Step title="Install Rust">
    Homebrew keeps rustup keg-only, so you need to add it to your path manually:

    ```bash
    brew install rustup
    echo "export PATH=\"$(brew --prefix rustup)/bin:\$HOME/.cargo/bin:\$PATH\"" >> ~/.zprofile
    exec zsh -l
    rustup toolchain install 1.93.0
    ```
  </Step>

  <Step title="Install required tools">
    This installs `cargo-nextest`, `cargo-watch`, and a nightly `rustfmt` build used by the formatter targets:

    ```bash
    make install-tools
    ```
  </Step>

  <Step title="Install the pre-commit hook">
    The hook runs `make fmt-check` and `make test` before each commit:

    ```bash
    make install-hooks
    ```
  </Step>
</Steps>

## Development commands

| Command | Purpose |
|---|---|
| `make fmt` | Auto-format code (requires nightly) |
| `make fmt-check` | Check formatting without modifying files |
| `make check` | Compile check — catches type and syntax errors |
| `make test` | Run unit tests |
| `make dev` | Fast local debug: build `kaku-gui` and run from `target/debug` |
| `make build` | Compile binaries (no app bundle) |
| `make app` | Build debug app bundle → `dist/Kaku.app` |

## Recommended workflow

Run this sequence before opening a pull request:

```bash
make fmt && make check && make test && make dev
```

To increase log verbosity during `make dev`, set `RUST_LOG`:

```bash
RUST_LOG=debug make dev
```

## Building a release

The `scripts/build.sh` script handles full release builds, including universal binary compilation and DMG creation:

```bash
# Full build + DMG (universal binary, release profile)
./scripts/build.sh

# Current architecture only — faster for local testing
./scripts/build.sh --native-arch

# App bundle only, skip DMG creation
./scripts/build.sh --native-arch --app-only

# Build and open the app automatically
./scripts/build.sh --native-arch --open
```

Output files are written to `dist/Kaku.app` and `dist/Kaku.dmg`.

## Pull requests

<Steps>
  <Step title="Fork and branch">
    Fork the repository on GitHub and create a branch from `main`.
  </Step>

  <Step title="Make your changes">
    Implement your fix or feature.
  </Step>

  <Step title="Verify locally">
    ```bash
    make fmt && make check && make test
    ```
  </Step>

  <Step title="Commit and push">
    Commit your changes and push the branch to your fork.
  </Step>

  <Step title="Open a pull request">
    Open a PR targeting `main`. CI will run in order: format check → unit tests → cargo check → universal build validation.
  </Step>
</Steps>
