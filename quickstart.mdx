---
title: "Get started with Kaku in minutes"
description: "Install Kaku on macOS, open it for the first time, and use the built-in AI assistant to generate your first shell command in just a few steps."
---

Kaku is designed to be productive from the moment you open it. This guide walks you through installation, first launch, basic navigation, and your first AI-assisted command. The whole process takes less than five minutes.

<Steps>
  <Step title="Install Kaku">
    Download the DMG from the [latest GitHub release](https://github.com/tw93/Kaku/releases/latest), open it, and drag **Kaku.app** to your Applications folder. Alternatively, install via Homebrew:

    ```bash
    brew install tw93/tap/kakuku
    ```

    <Note>
      The Homebrew formula is `tw93/tap/kakuku` — not `kaku`. There is an older, unrelated package named `kaku` on Homebrew that will install the wrong application.
    </Note>
  </Step>

  <Step title="Open Kaku">
    Launch Kaku from Applications or Spotlight. The app is notarized by Apple, so it opens without any security warning or Gatekeeper prompt.

    On first launch, Kaku automatically runs `kaku init` internally to set up your shell environment — it installs the `kaku` wrapper script and sources shell integration for zsh or fish. You do not need to run anything manually.
  </Step>

  <Step title="Learn the basic navigation shortcuts">
    Kaku uses familiar macOS shortcuts. The table below covers the most common actions:

    | Action | Shortcut |
    | :--- | :--- |
    | New tab | `Cmd + T` |
    | New window | `Cmd + N` |
    | Close tab or pane | `Cmd + W` |
    | Navigate tabs | `Cmd + Shift + [` / `]` |
    | Split pane vertical | `Cmd + D` |
    | Split pane horizontal | `Cmd + Shift + D` |
    | Open settings panel | `Cmd + ,` |
    | AI panel | `Cmd + Shift + A` |
    | Apply AI suggestion | `Cmd + Shift + E` |
  </Step>

  <Step title="Try the AI assistant">
    At any shell prompt, type a `#` followed by a plain-English description of what you want to do, then press **Enter**:

    ```bash
    # list all files modified in the last 7 days
    ```

    Kaku sends the description to your configured LLM and injects the resulting command back into the prompt for you to review before running. Nothing executes automatically.

    <Tip>
      When a command fails, Kaku automatically suggests a fix in the terminal. Press `Cmd + Shift + E` to apply it directly to the prompt.
    </Tip>
  </Step>

  <Step title="Verify your installation">
    Run the built-in health check to confirm that shell integration, environment variables, and the runtime are all working correctly:

    ```bash
    kaku doctor
    ```

    If everything is healthy, you will see a passing report. If anything is misconfigured, `kaku doctor` tells you exactly what to fix.
  </Step>
</Steps>
