The `kaku` command is the single entry point for managing every aspect of Kaku from your terminal. You can run `kaku` without arguments to open an interactive TUI menu, or pass a subcommand name to go directly to that action.

## Interactive menu

When you run `kaku` in an interactive terminal without a subcommand, Kaku renders a navigable menu:

```
  _  __      _
 | |/ /     | |
 | ' / __ _ | | __ _   _
 |  < / _` || |/ /| | | |
 | . \ (_| ||   < | |_| |  https://github.com/tw93/Kaku
 |_|\_\__,_||_|\_\ \__,_|  A fast, out-of-the-box terminal built for AI coding.

➤ 1. ai        Manage AI tools and Kaku Assistant
  2. config    Manage terminal and assistant settings
  3. init      Initialize shell integration
  4. doctor    Run diagnostics for shell and runtime health
  5. update    Check and install latest version
  6. reset     Remove Kaku shell integration and managed defaults

  ↑↓  |  Enter  |  1-6  |  Q Quit  |  Esc Exit
```

Navigate with arrow keys or `j`/`k`, select with Enter, jump directly with number keys `1`–`6`, or press the first letter of a command name. Press `Q` or `Esc` to exit without running anything.

## Subcommand summary

| Subcommand | Description |
|---|---|
| `kaku ai` | Configure Kaku Assistant and external AI coding tools |
| `kaku config` | Open `~/.config/kaku/kaku.lua` in your default editor |
| `kaku doctor` | Check shell integration, PATH entries, and runtime health |
| `kaku init` | Install shell integration and optional tools via Homebrew |
| `kaku update` | Download and install the latest Kaku release |
| `kaku reset` | Remove shell integration and restore managed defaults |
| `kaku cli` | Interact with the Kaku multiplexer from scripts |

<Note>
`kaku update`, `kaku reset`, and `kaku init` are currently supported on **macOS only**.
</Note>
